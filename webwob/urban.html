Membangun Game Interaktif Berbasis Sensor & Computer Vision

Dari Gerakan Tubuh ke Gameplay Real-Time 🎮

Bagaimana jika gerakan tubuh kita bisa langsung menjadi kontrol dalam sebuah game? Tidak perlu joystick, tidak perlu keyboard—cukup bergerak, dan sistem akan merespons secara real-time.

Itulah inti dari proyek ini: menggabungkan sensor, computer vision, dan komunikasi data real-time untuk menciptakan pengalaman bermain yang lebih imersif dan natural.

🎮 Lima Game, Lima Pendekatan

Dalam proyek ini, dikembangkan lima jenis game interaktif, masing-masing dengan pendekatan teknologi yang berbeda:

  • Shooting Game → menggunakan IMU dan ESP32 untuk mendeteksi arah dan gerakan
  • Tic-Tac-Toe → memanfaatkan kamera depth seperti Intel RealSense
  • Ping Pong → menggunakan kamera dan OpenCV untuk pelacakan bola
  • Running Game → berbasis pressure plate untuk mendeteksi langkah kaki
  • Swimming Game → menggunakan gyroscope dari smartphone

Setiap game dirancang untuk menerjemahkan gerakan fisik pemain menjadi input digital secara real-time, menciptakan interaksi yang jauh lebih natural dibandingkan kontrol konvensional.

🧠 Fokus Saya: Deteksi Bola & Komunikasi Sistem

Dalam keseluruhan proyek ini, saya berfokus pada dua bagian krusial di game ping pong: sistem deteksi bola dan komunikasi data antar sistem.

🟠 1. Program Deteksi Bola

Saya mengembangkan sistem berbasis computer vision menggunakan OpenCV untuk mendeteksi dan melacak pergerakan bola secara real-time.

Tantangan utama dalam bagian ini meliputi:

  • Membedakan bola dari background yang kompleks
  • Menjaga tracking tetap stabil saat bola bergerak cepat
  • Mengoptimalkan performa agar latency tetap rendah

Untuk mengatasi hal tersebut, saya menggunakan:

  • Color filtering / thresholding untuk isolasi objek
  • Contour detection untuk mengenali bentuk bola
  • Frame-by-frame tracking untuk menentukan arah dan kecepatan

Hasilnya adalah sistem yang mampu mendeteksi posisi bola secara konsisten dan cukup presisi untuk kebutuhan gameplay.

🔗 2. Komunikasi antara Deteksi dan Game

Selain deteksi, saya juga merancang mekanisme komunikasi data antara program computer vision dan game engine.

Tujuan utama sistem ini:

  • Mengirim posisi bola secara real-time
  • Menjaga sinkronisasi antara dunia nyata dan game
  • Meminimalkan delay agar gameplay tetap responsif

Pendekatan yang digunakan:

  • Pengiriman data posisi (x, y) secara kontinu
  • Format data ringan untuk kecepatan pemrosesan
  • Desain modular agar mudah diintegrasikan

Bagian ini sangat krusial—deteksi yang akurat tidak akan terasa jika komunikasi datanya lambat.

🚀 Kenapa Proyek Ini Menarik?

  • Embedded system (ESP32 & sensor)
  • Computer vision (OpenCV & kamera)
  • Real-time communication
  • Interactive system design

Dengan pendekatan ini, game tidak lagi terbatas pada controller tradisional, tetapi dapat langsung menggunakan gerakan tubuh sebagai input utama.

🌟 Penutup

Proyek ini menunjukkan bagaimana dunia fisik dan digital bisa dihubungkan secara real-time. Mulai dari deteksi objek menggunakan kamera hingga integrasi langsung ke gameplay tanpa delay yang mengganggu.

Ke depannya, konsep seperti ini memiliki potensi besar—tidak hanya untuk game, tetapi juga untuk olahraga interaktif, simulasi, dan berbagai aplikasi berbasis gerakan manusia.
